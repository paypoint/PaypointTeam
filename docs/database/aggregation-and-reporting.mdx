# 5. Aggregation and Reporting

Aggregation is one of the most important concepts in SQL.

In real-world applications, we rarely need raw data only.

Most business systems need reports like:

* Total users
* Total sales
* Average salary
* Highest order amount
* Monthly revenue
* Active customer count
* Department-wise employee count

Aggregation helps us summarize large datasets into meaningful insights.

This chapter teaches how SQL is used for reporting and analytical queries.

---

# 1. COUNT()

`COUNT()` is used to count rows.

It is one of the most commonly used aggregation functions.

---

# 1.1 COUNT(*)

Counts total rows in a table.

## Problem

Find total employees.

```sql
SELECT COUNT(*) AS total_employees
FROM employees;
```

## Output

| total_employees |
| --------------- |
| 6               |

---

# 1.2 COUNT(column_name)

Counts only NON-NULL values.

## Problem

Count how many employees have salary values.

```sql
SELECT COUNT(salary) AS total_salary_entries
FROM employees;
```

## Output

| total_salary_entries |
| -------------------- |
| 6                    |

---

# 1.3 COUNT with WHERE

## Problem

Count active employees.

```sql
SELECT COUNT(*) AS active_employees
FROM employees
WHERE is_active = TRUE;
```

## Output

| active_employees |
| ---------------- |
| 5                |

---

# 2. SUM()

`SUM()` adds numeric values.

Used heavily in:

* finance systems
* payroll systems
* reporting dashboards

---

## Problem

Find total salary expense.

```sql
SELECT SUM(salary) AS total_salary
FROM employees;
```

## Output

| total_salary |
| ------------ |
| 380000       |

---

## Problem

Find total salary of IT department.

```sql
SELECT SUM(salary) AS it_salary
FROM employees
WHERE department = 'IT';
```

## Output

| it_salary |
| --------- |
| 205000    |

# 6. GROUP BY

`GROUP BY` is used to create groups of rows.

Without `GROUP BY`, aggregation works on the entire table.

With `GROUP BY`, aggregation works per group.

This is one of the MOST IMPORTANT concepts in SQL reporting.

---

# 6.1 GROUP BY Department

## Problem

Find total employees in each department.

```sql
SELECT
    department,
    COUNT(*) AS total_employees
FROM employees
GROUP BY department;
```

## Output

| department | total_employees |
| ---------- | --------------- |
| IT         | 3               |
| HR         | 2               |
| Finance    | 1               |

---

# 6.2 GROUP BY with SUM()

## Problem

Find total salary department-wise.

```sql
SELECT
    department,
    SUM(salary) AS total_salary
FROM employees
GROUP BY department;
```

## Output

| department | total_salary |
| ---------- | ------------ |
| IT         | 205000       |
| HR         | 95000        |
| Finance    | 80000        |

---

# 6.3 GROUP BY with AVG()

## Problem

Find average salary department-wise.

```sql
SELECT
    department,
    AVG(salary) AS average_salary
FROM employees
GROUP BY department;
```

## Output

| department | average_salary |
| ---------- | -------------- |
| IT         | 68333.33       |
| HR         | 47500          |
| Finance    | 80000          |

# 7. HAVING

`HAVING` filters grouped data.

Important difference:

| Clause | Works On          |
| ------ | ----------------- |
| WHERE  | Individual rows   |
| HAVING | Aggregated groups |

---

# 7.1 HAVING Example

## Problem

Show departments having more than 1 employee.

```sql
SELECT
    department,
    COUNT(*) AS total_employees
FROM employees
GROUP BY department
HAVING COUNT(*) > 1;
```

## Output

| department | total_employees |
| ---------- | --------------- |
| IT         | 3               |
| HR         | 2               |

---

# 8. GROUP BY with ORDER BY

Aggregation results are often sorted for reporting.

---

## Problem

Show departments sorted by highest salary expense.

```sql
SELECT
    department,
    SUM(salary) AS total_salary
FROM employees
GROUP BY department
ORDER BY total_salary DESC;
```

## Output

| department | total_salary |
| ---------- | ------------ |
| IT         | 205000       |
| HR         | 95000        |
| Finance    | 80000        |

---

# 9. Multiple Aggregations Together

SQL allows multiple aggregation functions in one query.

---

## Problem

Generate salary report.

```sql
SELECT
    COUNT(*) AS total_employees,
    SUM(salary) AS total_salary,
    AVG(salary) AS average_salary,
    MIN(salary) AS minimum_salary,
    MAX(salary) AS maximum_salary
FROM employees;
```

## Output

| total_employees | total_salary | average_salary | minimum_salary | maximum_salary |
| --------------- | ------------ | -------------- | -------------- | -------------- |
| 6               | 380000       | 63333.33       | 45000          | 80000          |

# 11. Common Mistakes

---

# 11.1 Using WHERE with Aggregated Values

❌ Wrong

```sql
SELECT department, COUNT(*)
FROM employees
WHERE COUNT(*) > 1
GROUP BY department;
```

`WHERE` cannot filter aggregation results.

---

✅ Correct

```sql
SELECT department, COUNT(*)
FROM employees
GROUP BY department
HAVING COUNT(*) > 1;
```

---

# 11.2 Missing GROUP BY

❌ Wrong

```sql
SELECT department, COUNT(*)
FROM employees;
```

This causes error because non-aggregated column is used without grouping.

---

✅ Correct

```sql
SELECT department, COUNT(*)
FROM employees
GROUP BY department;
```

# 12. How to Solve Aggregation Problems

Always follow this mental checklist:

1. What report is required?
2. Which table contains data?
3. Which aggregation function is needed?
4. Is grouping required?
5. Is filtering needed before grouping (`WHERE`)?
6. Is filtering needed after grouping (`HAVING`)?
7. Is sorting required?

---

# Example Problem Breakdown

## Problem

"Find departments with more than 1 active employee and show total salary sorted by highest salary."

---

## Breakdown

Table → employees

Filter rows → is_active = TRUE

Group → department

Aggregation → SUM(salary), COUNT(*)

Filter groups → COUNT(*) > 1

Sort → total_salary DESC

---

## Final Query

```sql
SELECT
    department,
    COUNT(*) AS total_employees,
    SUM(salary) AS total_salary
FROM employees
WHERE is_active = TRUE
GROUP BY department
HAVING COUNT(*) > 1
ORDER BY total_salary DESC;
```
